Commute Compute System Architecture

A fully self-hosted smart transit display system for Australian public transport. Each user deploys their own complete stack with zero external dependencies. Features device pairing via Vercel KV for persistent configuration across serverless invocations.

v5.3 CCDash V12 LOCKED CCFirm v6.1-60s Vercel KV

Commute Compute Trademarked Tools

Four proprietary engines power the Commute Compute ecosystem, each handling a specific domain of the smart transit experience.

CCFirm™ v6.1 Firmware

The ESP32-C3 firmware that runs on TRMNL devices. CCFirm handles WiFi connectivity, HTTPS image fetching, BMP rendering, and partial display refresh. Includes device pairing via 6-character codes and anti-brick safety measures.

Algorithm Intelligence

Core algorithms that power intelligent journey planning.

Coffee Decision Algorithm

Source: src/core/coffee-decision.js

Inputs:
- Walking time: home cafe transit stop
- Next departure time (live GTFS-RT data)
- Cafe busyness level (time-based or Google Places)
- Coffee preparation time (2-8 min based on busyness)
- Safety buffer (2-3 min)
Decision Logic:
if (totalCoffeeTime + buffer) < minutesToDeparture GET COFFEE
else SKIP (go direct)

Cafe Hours: Automatically checks if cafe is open (default Mon-Sat 6am-5pm)

Cafe Busyness Detection

Source: src/services/cafe-busy-detector.js

Peak Times (Higher Wait):
- Morning Rush: 7-9am (Busy, ~6min wait)
- Lunch Rush: 12-2pm (Busy, ~5min wait)
- Afternoon Peak: 4-5pm (Moderate, ~4min wait)
- Off-Peak: Other times (Quiet, ~3min wait)

Uses Google Places API for live data when available, falls back to time-based estimation.

Smart Route Recommender

Source: src/services/smart-route-recommender.js

Route Patterns:
- Coffee + Tram + Train (multi-modal)
- Coffee + Train (direct with coffee)
- Train Direct (no coffee)
- Tram Only (when closer to tram stop)

Automatically discovers optimal routes based on your home/work/cafe locations.

Live Data Integration

Source: src/services/opendata-client.js

Data Sources:
- Transport Victoria OpenData API (GTFS-RT)
- Bureau of Meteorology JSON feeds
- Cache TTL: 30s for real-time, 5min for weather
Delay Detection:
Compares scheduled vs estimated departure times.
Categories: On-time (0-2min), Delayed (3-10min), Disrupted (>10min)
